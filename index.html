História dos computadores

voltar

Séc. XVII e Pascal
A história dos computadores é bem interessante porque mostra como a humanidade sempre
tentou facilitar os cálculos e resolver problemas complexos. Lá no século XVII, por exemplo,
Blaise Pascal criou a Pascalina, uma calculadora mecânica super simples, mas revolucionária
para a época. Depois, no século XIX, Charles Babbage teve uma ideia genial com o
"Motor Analítico", que é considerado o avô dos computadores modernos, embora ele
não tenha conseguido construir.

Séc. XX - ENIAC
As coisas começaram a pegar fogo no século XX, com as primeiras
máquinas que usavam eletricidade, como o ENIAC, criado depois da Segunda Guerra.
Esse foi um marco, porque, a partir daí, os computadores foram ficando menores,
mais rápidos e super importantes para o mundo. Dá pra ver que tudo começou de uma
curiosidade e criatividade humana, e hoje a gente não vive sem eles.

O que são Clients?

voltar

Definição:

Os clients são, basicamente, os "pedidos" que fazemos para acessar
informações ou serviços em um servidor. Eles são super importantes,
porque sem eles a internet não funcionaria como conhecemos. Quando
você entra em um site pelo navegador ou usa um app no celular, está
sendo um client, pedindo dados para um servidor, que devolve o que você precisa.

A ideia de clients é bem prática: eles são como o "meio" entre o usuário
e o servidor. O legal é que, com a evolução da tecnologia, os clients ficaram
mais inteligentes e rápidos, fazendo com que a experiência do usuário seja cada
vez mais fluida. Na minha opinião, entender como eles funcionam é essencial
para quem quer trabalhar com tecnologia, porque eles estão no centro de tudo o
que envolve comunicação digital.

O que são Servers?

voltar

Definição:

O server-side é o lado do servidor em uma aplicação ou site. É como o "cérebro" que processa tudo
o que acontece nos bastidores. Quando você faz uma ação no seu navegador ou app (o client), é o
server-side que recebe, interpreta e responde ao pedido. Ele cuida de coisas como acessar bancos
de dados, aplicar lógica de negócios e enviar os dados prontos para o client.

O server-side é essencial porque garante que os sistemas sejam organizados e funcionem corretamente.
Sem ele, não seria possível ter segurança nas transações, armazenar dados ou fazer operações complexas de forma eficiente.
Quem quer trabalhar com desenvolvimento precisa entender o server-side, porque é ali que muita mágica acontece para o usuário final.

Estrutura básica HTML

voltar

HTML 5
A estrutura básica do HTML é como o esqueleto de uma página web. Ela define a organização e os elementos essenciais
que dão forma ao conteúdo. Tudo começa com a tag , que envolve todo o código, e dentro dela temos duas partes
principais: o head e o body. O head cuida de coisas
nos bastidores, como título da página, links para estilos e scripts. Já o body é onde fica o conteúdo que o usuário vê, como textos, imagens, botões, etc.

Entender essa estrutura básica é o primeiro passo para quem quer aprender a criar sites. É simples, mas muito poderosa,
porque todo site, desde o mais básico até o mais avançado, começa com essa fundação. Com o tempo, dá para ir adicionando
estilos e funcionalidades, mas o HTML sempre será o ponto de partida.

Listas ordenadas (ol) e Não ordenadas (ul)

voltar

Lista ordenadas (ol)
As listas ordenadas no HTML são uma maneira super prática de organizar informações
em sequência, como um passo a passo ou uma lista de prioridades. Elas são feitas com a tag ol
(de ordered list) e cada item dentro dela é marcado com li (list item). O mais interessante
é que os itens são numerados automaticamente, o que facilita muito quando você precisa mostrar
algo em uma ordem específica.



Lista não ordenadas (ul)

As listas não ordenadas no HTML são usadas para organizar itens de forma simples
e sem uma ordem específica. Elas são definidas pela tag ul, e cada item da lista
é colocado dentro de uma tag li. O visual delas no navegador geralmente é com
marcadores (ou bullets), o que facilita a leitura e a compreensão dos itens.
